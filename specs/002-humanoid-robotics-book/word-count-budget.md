## Word Count Reduction Strategies

### Strategy 1: Code Listing Exclusion (Primary Strategy)

**Rationale**: The spec.md (FR-002) states "160,000-190,000 words **including code listings**". However, code listings are redundant with companion GitHub repositories. Readers can access full code online; book should show only essential snippets.

**Action**:
- **Before**: Include full code listings in chapters (e.g., 50-100 line Python scripts)
- **After**: Show only critical snippets (10-20 lines) with references to GitHub repo
- **Example**:
  ```python
  # Full 80-line code listing in chapter (current)
  class HumanoidController:
      def __init__(self, ...):
          ...
      def compute_trajectory(self, ...):
          ...
      # ... 60 more lines
  ```
  **Becomes**:
  ```python
  # Key snippet in chapter (revised)
  class HumanoidController:
      def compute_trajectory(self, target_pose):
          # Compute inverse kinematics
          joint_positions = self.ik_solver.solve(target_pose)
          return joint_positions

  # See full implementation: code/chapter-14-vla/src/controller.py
  ```

**Expected Reduction**: **30,000-35,000 words** (15-18% of current estimate)
- Assumptions:
  - Each hands-on chapter (18 chapters) currently has ~1,500-2,000 words of code listings
  - Reduction to ~300-400 words per chapter (80% reduction in code listings)
  - Total reduction: 18 chapters × 1,200 words saved = **21,600 words**
  - Conservative estimate: **30,000-35,000 words across all chapters and appendices**

**After Code Listing Reduction**:
- **Original Estimate**: 205,500-231,000 words
- **Revised (Code Reduction)**: 175,500-196,000 words
- **Target**: 160,000-190,000 words
- **Status**: **Within target range (185,000 midpoint) ✅**

### Strategy 2: Figure and Table Consolidation (Secondary Strategy)

**Rationale**: Reduce verbose figure captions and multi-page tables. Use concise captions with references to online resources for detailed data.

**Action**:
- **Before**: Multi-paragraph figure captions explaining every detail (200-300 words per complex figure)
- **After**: 1-2 sentence captions (30-50 words) + link to online detailed explanation
- **Before**: Multi-page tables with 20+ rows of data
- **After**: Summarized tables (top 5-10 rows) + link to full dataset in GitHub

**Expected Reduction**: **3,000-5,000 words**
- Assumptions: 50 complex figures/tables across book, reduce each by 50-100 words

---

### Strategy 3: Redundancy Elimination (Tertiary Strategy)

**Rationale**: Remove duplicated explanations across chapters. Use cross-references instead of repeating content.

**Action**:
- **Example**: ROS 2 topic publishing explained in Chapter 3, referenced (not re-explained) in Chapter 9, 10, 11, etc.
- **Before** (Chapter 9):
  ```markdown
  To publish a ROS 2 topic, you create a publisher object with create_publisher(),
  then call publish() with your message. The publisher sends the message to all
  subscribers on that topic. [150 words explaining ROS 2 publishing again]
  ```
- **After** (Chapter 9):
  ```markdown
  We publish detection results to `/detections` topic (see Chapter 3 for ROS 2
  topic publishing basics). [10 words + cross-reference]
  ```

**Expected Reduction**: **2,000-3,000 words**
- Assumptions: 10-15 instances of redundancy, reduce each by 150-200 words

---

### Strategy 4: Lab Instruction Streamlining (Quaternary Strategy)

**Rationale**: Simplify step-by-step instructions in hands-on labs. Assume reader has completed prior chapters and doesn't need every detail re-explained.

**Action**:
- **Before** (Chapter 11 Lab):
  ```markdown
  1. Open a terminal (Ctrl+Alt+T on Ubuntu).
  2. Navigate to your workspace: `cd ~/ros2_ws`
  3. Source ROS 2: `source /opt/ros/iron/setup.bash`
  4. Build the workspace: `colcon build --symlink-install`
  5. Source the workspace: `source install/setup.bash`
  [200 words for standard ROS 2 setup repeated in every lab]
  ```
- **After** (Chapter 11 Lab):
  ```markdown
  1. Set up ROS 2 workspace (see Chapter 2 for detailed steps).
  2. Clone the Nav2 config repo: `git clone ...`
  [30 words, cross-reference to Chapter 2]
  ```

**Expected Reduction**: **3,000-4,000 words**
- Assumptions: 18 hands-on labs, reduce each by 150-200 words of redundant setup instructions

## Success Criteria

**Primary Metric**: Final book word count (excluding front matter, index, bibliography) falls within **160,000-190,000 words**

**Secondary Metrics**:
1. No chapter exceeds its target range by >15% without justification
2. Code listings constitute <20% of total word count (down from ~25-30% without Strategy 1)
3. ≤5% of content is redundant (measured by cross-reference density: ≥20 cross-references per chapter)

**Validation Method**:
1. Automated word count script:
   ```bash
   # scripts/word-count.sh
   #!/bin/bash
   for file in book/chapters/part*/*.md; do
     wc -w "$file"
   done | awk '{total+=$1} END {print "Total: " total " words"}'
   ```
2. Manual review of 3 sample chapters per Part (18 chapters total) for redundancy and code listing density
3. Publisher review at draft submission (confirms page count estimate)
